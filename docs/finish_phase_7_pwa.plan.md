# Phase 7 – PWA & Offline

## Goal
The app works offline and can be "installed" on desktop and Android (and iOS via Safari).

## Steps

### Step 1: Add the Vite PWA plugin ✅
- Installed `vite-plugin-pwa` (uses Workbox under the hood).
- Configured in `vite.config.js` with `registerType: 'autoUpdate'` so the service worker auto-updates when new builds are deployed.
- Workbox `generateSW` strategy precaches all app assets (JS, CSS, HTML, SVG, PNG).

### Step 2: Configure the web app manifest ✅
- Manifest generated by the plugin at build time (`manifest.webmanifest`).
- Settings: `name`, `short_name`, `description`, `theme_color`, `background_color`, `display: standalone`, `orientation: portrait`.
- Icons: 192×192 PNG, 512×512 PNG, 512×512 maskable PNG.
- Enables "Install" in Chrome/Edge and "Add to Home Screen" on Android/iOS.

### Step 3: Create app icons ✅
- Source SVG: `public/icon.svg` (card-table themed: green felt circle, grid, sample cards, "CC" monogram).
- PNG icons generated via `scripts/generate-icons.mjs` (uses `sharp`):
  - `public/icon-192.png` (192×192)
  - `public/icon-512.png` (512×512)
  - `public/apple-touch-icon.png` (180×180)

### Step 4: Update HTML and register service worker ✅
- `index.html`: updated favicon to `icon.svg`, added `apple-touch-icon`, `apple-mobile-web-app-status-bar-style`, and `meta description`.
- `src/main.jsx`: dynamic import of `virtual:pwa-register` to register the service worker on app load.

### Step 5: Verify offline ✅
- `vite build` produces: `sw.js`, `workbox-*.js`, `manifest.webmanifest`, all icons, and precached assets.
- 15 precached entries (~276 KB).
- To test offline: `npm run build && npm run preview`, open in Chrome, then toggle DevTools → Application → Service Workers → Offline.

## Result
The game is a fully functional PWA. After the first visit, all assets are cached by the service worker and the game runs entirely offline with no network dependency.
